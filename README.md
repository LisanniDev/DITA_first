# Проекта на DITA
Шаблон и инструкция для создания документации в формате Docs-as-Code с использованием DITA.

## Возможности

* Работа в VS Code
* Хранение в Git
* Переиспользование контента
* Использование переменных
* Модульная структура
* Генерация брендированного PDF
* CI/CD с GitHub Actions

## Установка

### DITA-OT

Установка и настройка выполняется один раз.

1. Скачать DITA-OT с официального сайта: https://www.dita-ot.org/download
2. Распаковать скачанный архив в корень диска C (путь должен быть максимально коротким, не содержать кириллицы и пробелов).

### Java

1. Найти требуемую версию Java в описании README репозитория: https://github.com/dita-ot/dita-ot (**для DITA-OT 4.4 необходима Java 17**)
2. Скачать Java соответствующей версии с сайта: https://www.oracle.com/java/technologies/java-se-glance.html
3. Установить скачанный EXE-файл.

### Пути Java

Настройка путей выполняется один раз.

1. Нажать `Win+R`, ввести `sysdm.cpl` и нажать `ОК`.
2. В открывшемся окне во вкладке **Дополнительно** нажать **Переменные среды**.
3. В открывшемся окне в разделе **Системные переменные** нажать **Создать**.
4. Создать переменную:
   * имя: `JAVA_HOME`
   * значение: `путь-к-директории-java` (для Java 17 по умолчанию, например: `C:\Program Files\Java\jdk-17`)
5. Найти в системных переменных переменную `Path`.
6. В открывшемся окне найти или создать запись `%JAVA_HOME%\bin` и переменстить её в начало списка, нажимая **Вверх**.

### DitaCraft

1. В **VSCode** открыть раздел расширений **Extensions** (`Ctrl+Shift+X`).
2. В поиске расширений найти и установить **DitaCraft**.
3. Открыть настройки **DitaCraft** и указать в строке **Dita Ot Path** путь к распакованной ранее DITA-OT (например, `C:\dita-ot-4.4`).

## Подключение проекта

1. Открыть терминал в VS Code (`Ctrl+``).
2. Выбрать директорию для клонирования репозитория:
```
cd путь-к-директории
```
3. Клонировать репозиторий:
```
git clone https://github.com/your-company/название-репозитория.git
```
4. Открыть склонированную папку: `File - Open Folder`.

## Применение внесённых изменений на Git-репозиторий

Здесь описан пример применения изменений с GitHub.

1. Применить изменения в локальной директории (`Ctrl+S`).
2. Открыть панель **Source Control**.
3. Кратко описать внесённые изменения в строке `Message` (обязательно!).
4. Нажать кнопку **Commit**, а затем **Sync Changes**.

## Сборка PDF

### Сборка через **VSCode Tasks**

Для сборки PDF-файлов используется встроенный инструмент **VSCode Tasks**.
Доступны следующие команды:
* сборка pdf-файлов на все модели;
* сборка pdf-файлов на выбранные модели;
* очистка директории с pdf-файлами.

Процесс сборки через **VSCode Tasks**:

1. С открытым проектом нажмите `Ctrl+Shift+P` и очистите открывшуюся строку быстрого доступа.
2. Введите `task` и нажмите **Пробел**.
3. (ЛИБО вместо 1-2) Нажмите на строку быстрого доступа и выберите пункт **Run Task**.
4. Из выпадающего списка выбрать необходимую задачу:
   * DITA: Собрать ВСЕ (выполнение начинается сразу)
   * DITA: Собрать ВЫБРАННЫЕ (нажать `Enter`, ввести идентификаторы моделей через запятую без пробела, снова нажать `Enter`)
   * DITA: Очистить (нажать `Enter` для выполнения)
5. Проверить результат сборки в директории **out**.
