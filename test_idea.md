# Тестирование идей со стилями

Пробуем реализовать идею с внедрением DITA-OT в структуру проекта.

## Ожидаемый результат

* нет необходимости устанавливать DITA-OT
* установка плагина происходит в процессе сборки
* работает без привлечения админов
* необходима только установка VSCode и Java 17

## Структура проекта

```
DITA_first/
├─ .vscode
│  └─ tasks.json          <-- таска для запуска скриптов через панель VSCode
├─ dita-ot/               <-- копия DITA-OT в каждом проекте
├─ scripts/               <-- скрипты для tasks.json
│  ├─ utils.ps1           <-- формирование имени pdf-файла
│  ├─ build-all.ps1       <-- сборка всех main-карт проекта в pdf
│  ├─ build-selected.ps1  <-- сборка выбранных main-карт проекта в pdf (по идентификатору модели)
│  └─ clean.ps1           <-- очистка содержимого /out
├─ keys/                  <-- ключи-переменные
│  ├─ keys-base.ditamap   <-- общие переменные
│  ├─ keys-S2122I.ditamap <-- переменные, зависящие от модели устройства
│  └─ ...
├─ maps/                  <-- карты структуры документа
│  ├─ base.ditamap        <-- карта, определяющая порядок топиков
│  ├─ main-S2122I.ditamap <-- финальная карта для подключения переменных модели, стилей и структуры
│  └─ ...
├─ topics/                <-- разделы документа в формате топиков *.dita
├─ plugins/               <-- плагины для оформления документов по брендбуку
│  ├─ org.graviton.r4/    <-- плагин для руководства по эксплуатации формата А4
│  ├─ org.graviton.p5/    <-- плагин для паспорта формата А5
│  └─ .../    
└─ README.md              <-- описание проекта и инструкции
```

## Установка плагина

Перед сборкой проекта необходима установка плагина.

Для установки плагина вручную:
1. Открыть склонированный проект в VS Code.
2. Открыть терминал в VS Code (`` Ctrl+` ``).
3. Перейти в расположение DITA-OT в проекте:
```
cd dita-ot
```
4. Установить плагин:
```
.\dita.bat install ..\plugins\название-плагина
```

Если изменения в результате сборки не видны, то можно попробовать очистить кеш DITA-OT, указав в конце команды сборки `--clean-temp`.

Установку плагина обязательно выполнять только при:
* изменении `plugin.xml`;
* изменении структура фойлов плагина;
* добавлении новые xsl-файлы в плагин;
* клонировании проекта.

Для большего user-friendly проекта установка плагина встраивается в процесс сборки через таски VS Code и выполняется при каждой сборке.

В случае необходимости чистой переустановки плагина перед командой `install` выполнить:
```
.\dita.bat uninstall ..\plugins\название-плагина
```
